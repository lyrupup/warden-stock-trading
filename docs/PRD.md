## 1. 项目概述

### 1.1 项目简介

守望者股票交易系统（Warden Stock Trading System）是一套面向个人投资者的**线上 Web 端股票辅助决策与交易管理系统**。系统聚焦于「行情监控 + 策略管理 + 持仓记录 + 风控分析 + AI 决策辅助 + 自动化推送」的一体化闭环，帮助用户在每个交易日做到「盘前有预案、盘中有监控、盘后有复盘」。

系统强调 **AI 深度参与决策**：用户可以将自己的选股策略沉淀为可复用的 `skill.md` 文档，让大模型基于个人策略对具体个股进行专业化分析，并把分析结论按股票代码归档，形成个人投研知识库。

### 1.2 灵感来源

本项目的命名与功能设计灵感来源于《魔兽争霸 III》（Warcraft III）中的英雄角色 **守望者（Warden）**。守望者是一名以「侦查、洞察、警戒、追猎」著称的暗夜精灵刺客，象征着对目标的持续监控与精准打击。

我们将守望者的核心特质映射为系统的四大「技能彩蛋」（即四类核心能力），贯穿于各功能模块之中：

| 技能彩蛋 | 英文代号 | 对应系统能力 | 落地模块 |
|---------|---------|------------|---------|
| 🔍 侦查 | **Scan** | 拉取实时行情、持仓数据，对市场与账户进行持续探查 | 行情数据模块、持仓记录模块 |
| 🧠 洞察 | **Insight** | AI 深度分析、趋势判断、个股研判 | AI 对话分析模块、持仓分析与风控模块 |
| 🚨 警戒 | **Alert** | 异动风险提醒、大跌告警、风险预案 | 持仓分析与风控模块、定时任务模块 |
| 📨 传信 | **Message** | 飞书报告推送，把分析结论与告警送达用户 | 定时任务模块、个人中心配置模块 |

> 在产品交互与文案中，可适度使用「侦查 / 洞察 / 警戒 / 传信」作为功能区或操作按钮的彩蛋式命名，增强产品调性与记忆点。

### 1.3 产品定位

- **目标用户**：有一定经验、希望系统化管理自己投资流程的个人股民 / 量化爱好者。
- **核心价值**：把分散在 Excel、看盘软件、聊天群、AI 工具中的投研工作流，整合到一个可沉淀、可自动化、可复盘的个人系统中。
- **平台形态**：B/S 架构的线上 Web 系统，支持响应式布局（PC 优先，兼容平板 / 移动端浏览器）。

### 1.4 文档目标与范围

本 PRD 用于明确**产品功能需求与边界**，作为后续输出《前端技术开发文档》《后端技术开发文档》《数据库设计文档》《接口设计文档》的唯一需求依据。本文档**不限定**具体实现细节（具体技术选型仅作建议，见第 7 章）。

## 3. 用户与权限模型

### 3.1 用户体系（多用户预留）

> **重要约束**：当前阶段（V1）可以以「单用户 / 默认用户」方式上线使用，**但所有功能设计与数据模型必须为多用户场景预留扩展**，避免后期改造时大规模重构。

设计要求：

1. **数据归属**：所有业务数据（自选股、策略、持仓、交易记录、报告、定时任务、配置等）都必须携带 `user_id` 归属字段，逻辑上以用户维度隔离。
2. **登录认证**：预留标准的注册 / 登录 / 鉴权能力（账号密码、Token/Session）。V1 阶段可使用内置默认用户自动登录，但接口与鉴权中间件需具备开关式扩展能力。
3. **配置隔离**：用户的敏感配置（大模型 API Key、飞书 Key 等）按用户维度独立存储与加密。
4. **越权防护**：所有数据查询 / 修改接口都必须以当前登录用户为边界，禁止跨用户访问。

### 3.2 角色定义（V1 + 未来）

| 角色 | V1 是否启用 | 说明 |
|------|-----------|------|
| 普通用户（User） | 是 | 系统主要使用者，拥有自己的全部业务数据 |
| 管理员（Admin） | 预留 | 用户管理、系统配置、数据巡检（未来开放多用户后启用） |
| 访客（Guest） | 预留 | 只读体验（未来可选） |

## 5. 功能模块详细需求

> 说明：以下每个模块给出 **功能描述、核心功能点、关键数据对象、验收要点**。技术实现细节留待技术开发文档展开。

### M1 大盘与 A 股行情数据模块（Scan · 侦查）

#### 功能描述
为用户提供每日大盘概览、A 股行情数据，以及用户自选个股的行情拉取与展示，是整个系统的「数据眼睛」。

#### 核心功能点
- **F1.1 每日大盘行情自动展示**：进入系统首页 / 行情页自动展示当日大盘核心指数（上证指数、深证成指、创业板指、科创 50、沪深 300 等），含涨跌幅、成交量、成交额等。
- **F1.2 大盘市场概览**：展示市场情绪指标（涨跌家数、涨停 / 跌停数、北向资金等，依数据源能力裁剪）。
- **F1.3 自选股管理**：用户可添加 / 删除自选个股，分组管理，支持搜索（按代码 / 名称）。
- **F1.4 自选个股行情拉取**：展示自选股实时 / 准实时行情（现价、涨跌幅、量比、换手率等）。
- **F1.5 个股行情详情**：单只个股的 K 线（日 / 周 / 月）、分时、基础财务概要（按数据源能力）。
- **F1.6 行情数据刷新策略**：交易时段内按设定频率刷新；非交易时段展示收盘 / 最新快照。数据来源、刷新频率可配置。
- **F1.7 个股行情查询**：左侧「侦查」分区下提供独立的「个股行情」页面（与「行情中心」并列），用户输入股票代码 / 名称搜索，确认后展示该股**当日行情 + 历史 K 线图**，无需先加入自选即可快速查看任意个股。

#### 关键数据对象
- 指数行情（IndexQuote）：指数代码、名称、现价、涨跌额、涨跌幅、成交量、成交额、日期/时间。
- 个股行情（StockQuote）：股票代码、名称、现价、涨跌幅、开高低收、成交量额、换手率等。
- 自选股（Watchlist / WatchItem）：user_id、股票代码、分组、备注、添加时间。

#### 验收要点
- 交易日打开系统能自动看到当日大盘数据，无需手动操作。
- 自选股增删改查正确，数据按用户隔离。
- 行情数据源异常时有降级提示，不阻塞页面。

### M2 个人选股策略模块（Insight · 洞察）

#### 功能描述
用户可定义并管理属于自己的选股策略；每个策略不仅是文字描述，还包含**结构化、可量化的技术指标定义**与**可驱动 AI 的 skill.md 文档**，并支持**历史回测**验证策略有效性。

策略是「**量化粗筛 → AI 精筛**」两段式选股漏斗的方法论输入：

```
全市场/板块/自选池
     │  ① 量化指标粗筛（M2 本期核心，纯规则计算，无 AI）
     ▼
 候选股票池（命中量化条件的标的 + 各指标快照值）
     │  ② AI 精筛（后续：M6 加载策略 skill.md 逐只深度研判）
     ▼
 最终决策标的 + 结构化分析报告
```

> **本期（M2）范围聚焦第 ① 步「量化指标粗筛」**：把人脑里的盘面经验（均线多头排列、乖离、振幅分歧等）翻译成**可计算、可组合、可复用**的量化指标，由后台服务对股票池做规则化粗筛，产出候选列表。第 ② 步 AI 精筛复用 M6 能力（按候选逐只送入大模型 + 策略 skill.md），不在本期实现，仅打通数据交接口子。

#### 核心功能点
- **F2.1 策略管理（CRUD）**：新建 / 编辑 / 删除 / 复制策略，策略列表支持搜索与标签分类。
- **F2.2 策略描述**：富文本 / Markdown 描述策略思路、适用场景、风险偏好等。
- **F2.3 策略指标定义（量化）**：以**结构化、可组合**的方式定义选股指标与条件，支持基本面（PE、PB、ROE 等）与**技术面量化因子**（均线及多头/空头排列、乖离率、K 线振幅、连续振幅、N 日涨跌幅、量价、换手率等），支持因子间相互比较（如 `MA5 > MA10`）与多条件「与 / 或」嵌套组合。
- **F2.4 策略 skill.md 文档**：每个策略关联一份 `skill.md`（大模型提示词文档），定义 AI 在该策略下分析个股时应遵循的视角、步骤、输出格式。支持在线编辑与版本留存。
- **F2.5 策略回测**：选定历史区间与标的池，基于指标定义对策略历史表现进行回测，输出关键指标（收益率、最大回撤、胜率、夏普比率等）与回测曲线（后续迭代）。
- **F2.6 量化指标粗筛（本期核心）**：基于策略的量化指标定义，对指定股票池（全市场 / 指定板块 / 自选股 / 自定义代码列表）逐只计算技术因子并执行规则匹配，输出**命中条件的候选个股列表**（含命中标的的关键指标快照值与匹配评分）。粗筛为**纯规则计算、不调用大模型**，结果可落库、可导出、可一键加入自选。
- **F2.7 量化因子目录（通用能力）**：系统内置一套**可扩展的量化因子库**（见下「指标量化口径」），前端指标构造器据此动态渲染可选因子与参数，保证「指标定义 ↔ 粗筛引擎」共用同一套口径与 schema。
- **F2.8 粗筛 → AI 交接**：粗筛候选列表中的个股可一键发起 M6 AI 分析（携带本策略 + skill.md），完成「量化粗筛 → AI 精筛」的接力（AI 部分由 M6 落地）。

#### 指标量化口径（核心：把盘面经验翻译成可计算因子）

> 下列因子均基于个股**前复权日 K 线序列**（按时间升序，最新一根记为当前 bar `t`）计算，是粗筛引擎与指标构造器共用的统一口径。所有因子可作为规则的左/右操作数，与常量或另一个因子比较。

| 因子（type） | 含义 | 计算口径 | 取值 | 典型用法 |
|------|------|---------|------|---------|
| `ma` | N 日简单移动均线 | `MA(N) = 最近 N 根 close 之和 / N` | 数值 | 均线本身或与价/其他均线比较 |
| `ma_align` | 均线多头/空头**排列** | 给定有序周期 `[p1,p2,...]`，多头：`MA(p1) > MA(p2) > ...`（严格递减成立则真）；空头反之；可设容差 `eps`（默认 0） | 布尔 | `MA5>MA10>MA20`、`MA20>MA30>MA60` |
| `bias` | N 日**乖离率** | `BIAS(N) = (close_t − MA(N)) / MA(N) × 100%` | 数值（%） | 控制追高幅度（如 `bias5 ≤ 8%`），或捕捉超跌反弹 |
| `amplitude` | 当日 K 线**振幅** | `AMP_t = (high_t − low_t) / prev_close_t × 100%`（表征当日多空分歧剧烈程度） | 数值（%） | `amplitude > 7%` 表示当日分歧剧烈 |
| `amplitude_streak` | **连续振幅** | 从当前 bar 向前回看，连续满足 `AMP ≥ threshold` 的天数 `≥ days` 则真 | 布尔 | 「振幅大于某值连续多日」=震荡剧烈的中期票 |
| `pct_change` | N 日**涨跌幅** | `(close_t − close_{t−N}) / close_{t−N} × 100%` | 数值（%） | 区间动量过滤 |
| `field` | 原始行情字段 | 直接取当前 bar 的 `close/open/high/low/prev_close/volume/turnover_rate/change_percent` | 数值 | 价/量/换手等阈值过滤 |
| `vol_ratio` | 量比 | `volume_t / 最近 N 日均量`（可选，依数据源） | 数值 | 放量过滤 |
| `const` | 常量 | 用户给定常量值 | 数值/布尔 | 作为比较右操作数 |

**规则与组合（通用 schema）**：单条规则统一为「`left`（因子）`op`（操作符 `> >= < <= == !=`）`right`（因子或常量）」；布尔因子（`ma_align`/`amplitude_streak`）默认与「真」比较。规则可在「与/或」组（`logic: and|or`）中**嵌套**，从而表达任意复杂的复合条件。该 schema 同时用于「策略指标定义存储」「指标构造器渲染」「粗筛引擎执行」三处，保证三端一致。

#### 预置策略示例（开箱即用模板）

**策略一 · 短线均线多头排列（主升浪追高）** —— 短期趋势强势主升浪票，适合追高入场、破短期趋势止损离场：

```jsonc
// indicators（logic=and）
[
  { "left": {"type":"ma_align","periods":[5,10,20],"direction":"bull"}, "op":"==", "right":{"type":"const","value":true} }, // 短均线多头排列 MA5>MA10>MA20
  { "left": {"type":"field","field":"close"}, "op":">", "right":{"type":"ma","period":5} },                                  // 收盘价站上 MA5
  { "left": {"type":"bias","period":5}, "op":"<=", "right":{"type":"const","value":8} }                                      // MA5 乖离率 ≤ 8%，避免追太高
]
```

**策略二 · 中长线多头 + 短期剧烈震荡（波段/做 T）** —— 中期趋势向上但短期分歧剧烈的震荡票，适合波段与日内 T：

```jsonc
// indicators（logic=and）
[
  { "left": {"type":"ma_align","periods":[20,30,60],"direction":"bull"}, "op":"==", "right":{"type":"const","value":true} }, // 中长均线多头排列 MA20>MA30>MA60
  { "left": {"type":"amplitude_streak","threshold":5,"days":3}, "op":"==", "right":{"type":"const","value":true} }            // 振幅 ≥ 5% 连续 ≥ 3 日（剧烈震荡）
]
```

> 两个模板在系统中以「内置策略模板」形式提供，用户可一键复制为自己的策略后再调参（周期、阈值、连续天数等均可改）。

#### 关键数据对象
- 策略（Strategy）：user_id、名称、描述、标签、状态、创建/更新时间。
- 策略指标（StrategyIndicator）：量化因子规则组（`{logic, rules[], groups[]}`，JSONB）。
- 策略 skill 文档（StrategySkill）：strategy_id、skill.md 内容、版本号。
- 量化因子目录（IndicatorCatalog）：系统内置可选因子的类型、参数、说明（只读元数据，供前端构造器渲染）。
- 粗筛任务 / 结果（ScreenTask / ScreenResult）：user_id、strategy_id、股票池范围、交易日、状态、扫描数量、命中数量、候选列表（每只含命中指标快照值与评分）、错误信息。
- 回测任务 / 结果（BacktestTask / BacktestResult）：参数、区间、结果指标、曲线数据（后续迭代）。

#### 验收要点
- 策略与其 skill.md、量化指标定义一一关联，可被 M6 引用。
- **量化粗筛可对指定股票池运行，正确按因子规则匹配，产出含指标快照值的候选列表**；同一份指标定义在「构造器渲染」与「粗筛执行」结果一致。
- 因子计算口径（均线/排列/乖离/振幅/连续振幅）有明确公式并经单元测试覆盖。
- 粗筛候选可加入自选、可一键交接 M6 发起 AI 分析。
- skill.md 可编辑、可保存、可在 AI 分析时被加载；回测产出核心指标（后续迭代）。

### M3 个人持仓记录模块（Scan · 侦查）

#### 功能描述
用户记录与管理自己的真实 / 模拟持仓；通过录入买卖交易记录，系统**自动计算每笔及整体的盈亏情况**，形成清晰的账户视图。

#### 核心功能点
- **F3.1 持仓管理**：新建 / 编辑 / 关闭持仓（一个持仓对应一只股票的仓位汇总）。
- **F3.2 交易记录录入**：在持仓下记录买入 / 卖出操作（成交价、数量、手续费、税费、时间、备注）。
- **F3.3 自动盈亏计算**：
  - 持仓成本与持仓均价（支持移动加权 / 先进先出口径，口径可在技术文档中明确，默认移动加权平均）。
  - 浮动盈亏（结合 M1 实时行情）与已实现盈亏（基于卖出记录）。
  - 单笔盈亏、单只累计盈亏、账户总盈亏与收益率。
- **F3.4 账户总览**：总市值、总成本、总浮动盈亏、总收益率、当日盈亏。
- **F3.5 持仓与行情联动**：持仓个股自动接入 M1 行情，实时刷新市值与浮盈。
- **F3.6 交易流水查询**：按时间 / 股票 / 买卖方向筛选历史交易记录。

#### 关键数据对象
- 持仓（Position）：user_id、股票代码、持仓数量、持仓均价、成本、状态（持有/已清仓）。
- 交易记录（Trade）：position_id、方向（买/卖）、价格、数量、费用、成交时间、盈亏（卖出时计算）。

#### 验收要点
- 录入买卖后，持仓数量、均价、盈亏自动正确计算。
- 已清仓持仓与在持持仓状态清晰区分。
- 盈亏计算口径有明确说明且可复核。

### M4 持仓分析与风控模块（Insight 洞察 + Alert 警戒）

#### 功能描述
基于持仓数据 + 行情数据 + AI 能力，对用户账户进行**每日持仓分析、风险提示与明日盘前预案生成**，帮助用户为下一交易日提前备案。

#### 核心功能点
- **F4.1 每日持仓 AI 分析**：一键 / 定时触发，AI 结合当前持仓、行情、市场环境，输出持仓健康度分析（结构、集中度、行业分布、个股点评等）。
- **F4.2 风险提示**：识别并提示风险点，如：单一持仓过度集中、个股大幅回撤、止损线触发、行业系统性风险、流动性风险等，并给出风险等级。
- **F4.3 异动 / 大跌告警**：当持仓个股出现异动（大涨大跌、触及预设阈值）时生成告警（与 M5 定时任务、Message 传信联动推送）。
- **F4.4 明日盘前预案**：针对每只持仓生成下一交易日的应对预案（关注价位、止盈止损建议、加减仓思路、关注事件），形成「盘前作战计划」。
- **F4.5 分析报告归档**：每日分析与盘前预案保存为报告，支持按日期回看。

#### 关键数据对象
- 持仓分析报告（PositionAnalysisReport）：user_id、日期、分析内容、风险项列表、风险等级。
- 盘前预案（PreMarketPlan）：user_id、交易日、按个股的应对策略。
- 告警记录（AlertRecord）：触发条件、对象、级别、时间、推送状态。

#### 验收要点
- 能基于真实持仓产出有针对性的分析（非泛泛而谈）。
- 风险提示有明确级别与对应建议。
- 盘前预案可保存、可回看、可推送。

---

### M5 个人定时任务模块（Alert 警戒 + Message 传信）

#### 功能描述
用户可自定义定时任务，实现**自动化运行**：定时拉取数据、定时跑分析、定时出报告、定时推送到飞书（云文档 / 机器人）。这是系统「自动化引擎」与「传信」能力的载体。

#### 核心功能点
- **F5.1 定时任务管理（CRUD）**：创建 / 编辑 / 启用 / 停用 / 删除定时任务。
- **F5.2 调度配置**：支持 Cron 表达式或可视化周期配置（每日盘前 / 盘后、每周、自定义时间等），支持交易日历感知（仅交易日执行，可选）。
- **F5.3 任务类型**：
  - 定时脚本 / 数据拉取任务（如盘后拉取行情快照）。
  - 定时分析任务（如每日盘后生成持仓分析报告、明日盘前预案）。
  - 定时报告生成任务（汇总当日行情、持仓、AI 结论）。
  - 定时推送任务（将报告推送到飞书云文档 / 飞书机器人）。
- **F5.4 推送渠道**：对接飞书机器人 Webhook（消息卡片 / 文本）与飞书云文档（写入 / 更新文档），凭证取自 M7 配置。
- **F5.5 执行记录与日志**：记录每次任务执行的时间、状态（成功/失败）、输出与错误信息，支持重试。
- **F5.6 手动触发**：定时任务支持立即手动运行一次（便于调试）。

#### 关键数据对象
- 定时任务（ScheduledTask）：user_id、名称、类型、调度表达式、目标动作配置、启用状态。
- 任务执行记录（TaskExecutionLog）：task_id、开始/结束时间、状态、结果/错误。

#### 验收要点
- 定时任务能按配置周期自动触发并执行。
- 报告能成功推送到飞书机器人 / 云文档。
- 执行失败有日志记录且可重试 / 手动触发。

---

### M6 AI 对话分析模块（Insight · 洞察）

#### 功能描述
系统的「智能投研助手」。用户选定**自己名下的某个选股策略**（携带其 skill.md），对**某只具体个股**发起 AI 分析对话，AI 基于策略方法论 + 个股行情数据输出**结构化分析报告**；报告**按股票代码集中归档**，便于后续回看与对比。

#### 核心功能点
- **F6.1 策略 + 个股选择**：用户选择一个已有策略 + 输入 / 选择一只个股，发起分析。
- **F6.2 AI 分析对话**：以对话形式进行分析，AI 加载该策略的 skill.md 作为分析框架，并可引用 M1 行情、M3 持仓等上下文数据。
- **F6.3 分析报告生成**：输出结构化分析报告（结论、依据、关键指标、风险、操作建议等），格式由策略 skill.md 约定。
- **F6.4 报告按股票代码归档**：报告集中存储，以**股票代码**为主索引建立个人投研知识库，支持按代码 / 策略 / 日期检索与回看。
- **F6.5 多轮追问**：用户可在分析基础上继续追问、补充上下文。
- **F6.6 报告复用与导出**：报告可被 M5 推送到飞书，可导出 / 收藏。

#### 关键数据对象
- 分析会话（AnalysisConversation）：user_id、strategy_id、股票代码、消息列表。
- 分析报告（StockAnalysisReport）：user_id、股票代码、strategy_id、报告内容、生成时间、标签。
- 报告索引（按股票代码聚合）。

#### 验收要点
- 选定策略后 AI 分析明显体现该策略 skill.md 的视角与输出格式。
- 报告按股票代码归档且可检索回看。
- AI 调用使用的是 M7 中用户配置的大模型凭证。

### M7 个人中心配置模块（基础配置）

#### 功能描述
用户的个人设置中心，管理与隔离用户级别的**敏感配置与偏好设置**，类似「个人环境变量」。为其他模块提供凭证与偏好。

#### 核心功能点
- **F7.1 大模型配置**：配置大模型 API Key、Base URL、模型名称、默认参数（温度等），支持多家供应商 / 多套配置。
- **F7.2 飞书配置**：配置飞书机器人 Webhook、飞书应用 App ID / App Secret、云文档相关凭证（用于 M5 推送）。
- **F7.3 行情数据源配置**：行情数据源选择 / API Key（如使用第三方数据源），刷新频率偏好。
- **F7.4 主题与偏好**：白天 / 暗夜模式（Light/Dark）切换，语言、首页默认视图等偏好设置。
- **F7.5 个人资料**：昵称、头像等基础信息（为多用户预留）。
- **F7.6 安全存储**：所有密钥类配置加密存储，前端展示脱敏（如仅显示后四位），按用户隔离。

#### 关键数据对象
- 用户配置（UserConfig / UserSettings）：user_id、配置项键值（分组：AI / 飞书 / 数据源 / 偏好），敏感字段加密。

#### 验收要点
- 配置项保存后，AI 分析、飞书推送等功能能正确读取并生效。
- 密钥脱敏展示、加密存储。
- 主题切换即时生效并持久化。

## 6. 非功能性需求

| 类别 | 需求说明 |
|------|---------|
| 安全性 | 用户敏感配置（API Key 等）加密存储；接口需鉴权；防越权访问；防 XSS / CSRF / SQL 注入 |
| 多用户扩展 | 所有数据携带 user_id，逻辑隔离；鉴权中间件可平滑开启多用户 |
| 性能 | 行情页常用接口响应 < 1s（缓存命中场景）；AI 分析支持异步 / 流式返回 |
| 可用性 | 数据源 / AI / 飞书等外部依赖异常时有降级与友好提示，不阻塞核心页面 |
| 可观测性 | 关键操作与定时任务有日志；任务执行可追踪、可重试 |
| 数据合规 | 行情数据来源合法合规；个人投资数据仅用户本人可见 |
| 兼容性 | 主流现代浏览器；响应式布局（PC 优先，兼容平板 / 移动浏览器） |
| 可维护性 | 前后端分离；模块解耦；配置通过环境变量注入，不硬编码 |
| 国际化 | 预留 i18n 能力（V1 以中文为主） |

> ⚠️ 风险与免责：本系统提供的 AI 分析、风险提示、盘前预案等均为**辅助决策信息，不构成投资建议**。需在产品中明确风险提示文案。

## 7. 技术方案建议（非强制，供技术文档参考）

> 以下仅为建议性技术选型，最终以《前端 / 后端技术开发文档》为准。

- **整体架构**：前后端分离，B/S Web 系统；后端提供 RESTful API。
- **前端**：React + Vite + shadcn/ui + Tailwind CSS；数据请求 ky + TanStack Query；状态管理 zustand；支持 Light/Dark 主题；图表用于行情 K 线与回测曲线。
- **后端**：Go + Gin + GORM 作为核心业务服务；API 实现限流、超时中断（context 传播）、鉴权、日志、CORS 等中间件。
- **AI 网关**：统一封装大模型调用层，支持多供应商、流式输出，读取用户级配置。
- **定时调度**：后端内置任务调度（如 cron 调度器），支持任务执行记录与重试。
- **数据存储**：PostgreSQL（业务数据）+ Redis（行情缓存 / 会话 / 限流）；消息 / 异步任务可选 RabbitMQ。
- **基础设施**：PostgreSQL / Redis / RabbitMQ 通过 Docker 启动，docker-compose 统一编排；配置经环境变量注入。
- **外部集成**：行情数据源（第三方行情 API）、飞书开放平台（机器人 Webhook / 云文档 API）、大模型 API。

## 8. 数据模型概览（逻辑层，预留多用户）

> 仅为逻辑实体概览，物理表结构详见后续《数据库设计文档》。所有业务实体均含 `user_id`。

```
User(用户)
 ├─ Watchlist(自选股) ──< WatchItem
 ├─ Strategy(选股策略)
 │    ├─ StrategyIndicator(量化因子规则组)
 │    ├─ StrategySkill(skill.md 文档)
 │    ├─ ScreenResult(量化粗筛结果, 候选股票池)
 │    └─ BacktestResult(回测结果)
 ├─ Position(持仓) ──< Trade(交易记录)
 ├─ PositionAnalysisReport(持仓分析报告)
 ├─ PreMarketPlan(盘前预案)
 ├─ AlertRecord(告警记录)
 ├─ ScheduledTask(定时任务) ──< TaskExecutionLog(执行记录)
 ├─ AnalysisConversation(AI 分析会话) ──< StockAnalysisReport(分析报告，按股票代码索引)
 └─ UserConfig(个人配置：AI/飞书/数据源/偏好)

公共数据（不归属单一用户）：
 ├─ IndexQuote(指数行情)
 ├─ StockQuote(个股行情)
 ├─ IndicatorCatalog(量化因子目录, 内置只读元数据)
 └─ StockIndicatorSnapshot(个股技术指标快照, 可选, 供全市场粗筛加速)
```
